# 📦 Складський облік (геотекстиль + волокно)

Веб-програма зі спільною базою даних: одне посилання для комп'ютера і телефона.

## Розгортання на Railway (один раз, ~10 хвилин)

### Крок 1. Завантажте проєкт на GitHub
1. Зайдіть на github.com (зареєструйтесь, якщо нема акаунту)
2. Натисніть **+** (вгорі праворуч) → **New repository**
3. Назва: `sklad` → оберіть **Private** → **Create repository**
4. На сторінці репозиторію: **uploading an existing file** → перетягніть УСІ файли з цієї папки
   (server.js, package.json, README.md і папку public з усім вмістом) → **Commit changes**
   * Увага: якщо папка public не завантажилась перетягуванням — завантажте її файли окремо
     через Add file → Upload files, попередньо створивши шлях public/

### Крок 2. Розгорніть на Railway
1. Зайдіть на railway.com → **New Project** → **Deploy from GitHub repo** → оберіть репозиторій `sklad`
   (при першому разі Railway попросить доступ до GitHub — дозвольте)
2. Додайте базу даних: у проєкті натисніть **+ Create** → **Database** → **Add PostgreSQL**
3. Відкрийте сервіс `sklad` → вкладка **Variables** → додайте дві змінні:
   * `DATABASE_URL` = натисніть «Add Reference» і оберіть `Postgres.DATABASE_URL`
   * `APP_PASSWORD` = ваш пароль для входу в програму (придумайте)
4. Вкладка **Settings** → розділ **Networking** → **Generate Domain**
   → отримаєте посилання типу `sklad-production.up.railway.app`

### Крок 3. Користуйтесь
* Відкрийте посилання → введіть пароль → оберіть склад
* На айфоні: відкрийте посилання в Safari → Поділитися → **На Початковий екран** —
  буде іконка як у звичайного застосунку
* Усі пристрої бачать одні й ті самі залишки

## Перенесення ваших даних
Якщо ви вже вели облік у старій HTML-версії:
1. У старій програмі натисніть **💾 Резервна копія** — збережеться файл .json
2. У новій (на сайті) натисніть **📂 Відновити** і виберіть цей файл —
   дані запишуться на сервер і стануть видні всім пристроям

Якщо старих даних нема — при першому відкритті завантажаться залишки
з ваших файлів від 10.06.2026.

## Технічне
* Node.js + Express, дані в PostgreSQL (таблиця sklad, JSON на кожен склад)
* Без DATABASE_URL дані пишуться у файл data.json (для локального тесту)
* Локальний запуск: `npm install && APP_PASSWORD=test node server.js` → http://localhost:3000
